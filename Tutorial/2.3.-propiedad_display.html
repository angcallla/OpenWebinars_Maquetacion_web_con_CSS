Maquetación tradicional con CSS: Propiedad display

Como sabemos, los elementos en la web con interpretados como cajas con sus componentes (content, padding, margin). Estos tienen comportamientos en línea o en bloque que vienen determinados o por defecto o con css utilizando la propiedad "display"

Display: valores más usados

  • inline
  • inline-block
  • block
  • none
  • Relacionados con tablas
  • Flex y Grid (próximos cursos)

Y algunos más: enlace

Inline, Inline-block

Inline

Los elementos Inline no rompen el flujo de la línea y se van colocando uno detrás de otro mientras caben. Aceptan "margin" y "padding" (solo en horizontal). Ignoran "width" y "height".

Etiquetas inline por defecto: span, a, b, img (esta si hace caso de width y height), ...

Inline-Block

Funciona igual que Inline, pero si hace caso a las propiedades "width" y "height".

Block

Los elementos en bloque rompen el flujo de la línea y provocan "un salto de línea" tanto anterior como posterior. Por defecto si no especificamos una anchura, ocupan la del elemento que los contiene (la etiqueta padre).

Etiquetas block por defecto: h1 a h6, p, section, div, ul, li, nav, todas las etiquetas semánticas, ...

None

Una vez fijado el elemento desaparece. No dejará un espacio vacío aunque siga en el código HTML. Es diferente de la propiedad visibility:hidden que si deja el hueco aunque no se muestre.

Valores de display y relacionados con tablas

  • table (la etiqueta se va a comportar como una tabla)
  • table-cell (se comportará como un td o th)
  • table-row (se comportará como un tr)
  • table-caption
  • table-column
  • table-colgroup
  • table-header-group
  • table-footer-group
  • table-row-group
